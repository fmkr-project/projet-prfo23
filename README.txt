Utiliser une commande make classique afin de construire les deux exécutables.

main (issu de main.ml) représente la phase 1 (recherche de plus courts chemins).
main2 est utilisé durant la phase 2 (flot maximal).
Ces deux programmes nécessitent un fichier en entrée pour pouvoir fonctionner.
	ex: ./main2 test

A et B sont des scripts qui ont servi pendant le développement en tant que raccourci de ./main < input1 (resp. ./main2 < input2).
Ils peuvent être exécutés afin d'avoir un premier aperçu du bon fonctionnement du programme.
